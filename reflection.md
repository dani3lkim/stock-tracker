# Written Reflection

This app tracks a personal stock watchlist and investment portfolio, including live price data and gain/loss calculations. I chose this topic because I follow the stock market and wanted a tool I could actually use day to day. The hardest part was understanding how Streamlit forms work — specifically that validation messages and success alerts have to go outside the `st.form()` block, not inside it, which caused a "Missing Submit Button" error on deployment that took time to diagnose. Mid-project, I had to restructure the entire app from a single-file design into a multi-page app with a `pages/` folder, which I hadn't planned for originally. I also added a `tags` table and `stock_tags` junction table to meet the many-to-many requirement — my original design only had two tables and didn't account for this relationship at all. I also discovered that hardcoding the database URL directly in the code is a serious security issue, so I had to move it to Streamlit's secrets manager before deploying to a public GitHub repository. If I had another week, I would add price alerts that notify when a stock crosses a target threshold, and a historical performance chart showing the full portfolio value over time.
